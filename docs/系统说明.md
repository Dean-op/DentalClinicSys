# 口腔诊所管理系统说明

## 角色与边界

系统采用单前端、多角色路由的方式实现。登录成功后，前端根据后端返回的 `role` 跳转到患者端、医生端或管理员端；后端使用 Spring Security 和 JWT 在接口层再次校验角色。

## 功能覆盖

- 患者端：注册登录、公告浏览、医生查询、药品购买、在线预约、订单管理、预约管理、病例查看、AI牙医、症状推荐、用药提醒、留言评价。
- 医生端：登录、个人资料维护、排班维护、预约处理、接诊病例、电子处方、患者历史查询、留言回复、个人统计。
- 管理员端：用户管理、医生审核、药品管理、公告管理、病例监督、处方查看、订单处理、预约数据、统计分析、系统日志。

## AI牙医实现

AI牙医使用外部 OpenAI 兼容接口实现。患者输入症状后，后端会先读取 `symptom_rule` 规则表，匹配本地推荐科室、医生和药品，再把这些参考资料与患者症状一起发送到外部 `/chat/completions` 接口，由模型生成最终问诊建议。

配置位置在本地数据库 `ai_config` 表，不要把真实 `api_key` 写入 `application.yml` 或提交到远程仓库。

```sql
UPDATE ai_config
SET base_url = 'https://api.openai.com/v1',
    api_key = 'your_api_key',
    model = 'gpt-4o-mini',
    enabled = 1,
    updated_at = NOW()
WHERE id = 1;
```

如果使用第三方 OpenAI 兼容网关，只需要把 `base_url` 改成该网关的 `/v1` 地址，模型名改成网关支持的模型。

`data.sql` 只会初始化一条空密钥配置，并且重复导入时不会覆盖本地已填写的 `api_key`。

所有 AI 输出都带有提示：结果仅为初步参考，不能替代医生诊断。

## 数据初始化

后端启动时执行：

- `backend/src/main/resources/schema.sql`
- `backend/src/main/resources/data.sql`

新电脑开发推荐直接使用：

- `scripts/bootstrap-dev/bootstrap-dev.cmd`
- `scripts/bootstrap-dev/bootstrap-dev.ps1`

这套脚本会自动建库、导入初始化 SQL，并按“先后端、后前端”的顺序启动项目。

默认账号：

- 管理员：`admin / 123456`
- 医生：`doctor_chen / 123456`
- 患者：`patient_li / 123456`

## 演示流程

1. 患者登录后查看公告、医生和药品。
2. 患者输入症状使用 AI牙医，并提交预约。
3. 医生登录后确认预约，进入接诊，保存病例并开具处方。
4. 患者查看病例和用药提醒，也可以购买药品并申请退款。
5. 管理员登录后审核医生、维护药品公告、处理订单、查看统计和系统日志。

## 用药提醒说明

- 用药提醒按“患者 + 药品”聚合展示，同一种药只保留一条当前提醒。
- 默认只对“已用完”“今天用完”“2天内将用完”的药品给出补药提醒，避免页面出现大量无效预警。
- 患者端不展示库存状态，提醒文案仅围绕复诊与补药建议。
